事件级市场情报系统蓝图

DeepAlpha 把 X 信息流、历史推文、关系图谱和中文证据报告组合成一个可审计的市场情报系统。 它不追求“抓得更多”，而是追求“噪音更少、证据更硬、判断更透明”。

Intelligence Pipeline

系统不把单条推文直接当成结论，而是把问题拆解、动态抓取、六层清洗、事件聚类和证据链串成可审计的判断流程。

01

自然语言问题

油价会涨吗？BTC 会涨吗？SUI 是否有催化？

02

问题理解

识别资产、时间窗口、市场阶段和核心变量。

03

动态信号规划

按问题选择账号、关键词、历史库和候选冲击源。

04

X 情报抓取

从实时推文、历史 CSV、账号池和主题网络取数。

05

六层清洗

去重、低价值过滤、二手转述标记和情绪噪音识别。

06

事件聚类

把推文合并成事件簇，避免同源扩散重复计票。

07

关系图谱

连接账号、资产、事件、引用和共同出现关系。

08

证据链

区分原始信源、独立确认、冲击源和市场变量。

09

中文证据报告

给出方向判断、关键证据、风险提示和免责声明。

Signal Doctrine

DeepAlpha 的设计原则是把“内容热度”降权，把“事件证据”升权。每一条进入报告的内容都要通过来源、独立性、相关性和风险解释的过滤。

01

问题先行

先理解用户在问什么，再决定抓谁、抓什么，而不是把固定账号池重复套到所有问题上。

02

来源分层

区分原始信源、媒体转述、交易员观点、喊单噪音和机器人扩散。

03

事件聚合

相似推文先合并成事件簇，再判断它是否与资产方向有关。

04

证据可审计

报告必须说明支持、反对、冲突和不确定性，而不是只给一个方向标签。

Quality Gates

v0.1 稳定版的目标不是做炫，而是确保异常路径不崩、报告路径可用、关键证据能被检查。

Runtime

基础命令、JSON 输出、Markdown 格式化和模块入口必须稳定。

Crawl

账号不存在、抓取 0 条、历史推文为空时，系统必须可回退。

Report

中文解释、关键证据、风险提示和免责声明必须出现在最终报告里。

Debug

Claude X 报错、API 失败、清洗异常和抓取异常必须进入日志。
